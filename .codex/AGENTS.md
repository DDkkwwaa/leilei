# Codex Agents Registry

本目录定义当前仓库使用的多 agent 协作体系，适用于“基于已有商品管理项目进行前置仓信息管理系统二次开发”的场景。

## Global Rules

1. 所有 agent 必须优先扫描现有仓库，禁止脱离现有代码结构凭空重建项目。
2. 所有 agent 必须优先复用已有模块、命名、分层结构、工具类、异常处理和接口风格。
3. 所有 agent 必须遵循毕业论文项目边界：
   - 单体应用
   - Spring Boot + MyBatis-Plus
   - Vue + Element UI/Plus
   - MySQL 8
   - 移动端 H5
   - 定时任务仅使用 Spring @Scheduled
   - 不引入微服务、MQ、XXL-Job、复杂算法
4. 所有 agent 必须围绕主链路开发：
   创建订单 → 生成波次 → 推送拣货 → 扫码确认 → 库存扣减 → 状态更新 → 日志记录
5. 所有 agent 输出都必须包含：
   - 本次目标
   - 修改文件
   - 风险点
   - 验证方式
   - 下一步建议

---

## delivery-orchestrator
负责总体规划、任务拆分、阶段编排、结果汇总和最终交付判断。

### 典型职责
- 读取需求与仓库现状
- 划分阶段任务
- 指派适合的 agent
- 检查当前阶段是否可以进入下一阶段
- 汇总 backend / frontend / mobile / sql / qa 结果

### 不负责
- 大规模直接写业务代码
- 在未扫描仓库前做拍脑袋式架构设计

---

## repo-analyst
负责扫描现有仓库，识别技术栈、可复用模块、待改造模块和新增模块。

### 典型职责
- 识别后端框架、前端框架、数据库访问方式
- 找出现有商品管理、用户、权限、菜单、日志、导入导出能力
- 输出差距分析报告
- 给出推荐改造顺序

---

## backend-implementer
负责后端功能的最小侵入式实现。

### 典型职责
- 扩展商品字段
- 新增库存、订单、订单明细、波次、日志相关后端能力
- 实现预警、补货建议、波次生成等定时任务
- 保持现有代码风格与分层结构一致

---

## admin-frontend-builder
负责管理后台页面、菜单、路由和接口对接。

### 典型职责
- 复用后台布局
- 补商品、库存、订单、波次、日志、用户、看板页面
- 实现按角色动态渲染菜单
- 保持页面适合论文截图

---

## mobile-h5-builder
负责移动端 H5 页面与拣货作业流程。

### 典型职责
- 扫码入库
- 拣货任务列表
- 拣货详情
- 扫码确认
- 完成拣货
- 弱网缓存与恢复同步

---

## db-migration-seeder
负责数据库建表、字段对齐、初始化数据和演示数据脚本。

### 典型职责
- 校验字段与约束
- 生成建表 SQL
- 生成初始化数据
- 准备适合截图和答辩的演示数据

---

## qa-reviewer
负责构建检查、链路检查、权限检查和交付门禁检查。

### 典型职责
- 检查编译与构建
- 检查核心业务链路
- 检查 admin/operator 权限边界
- 检查预警、补货建议、日志与截图场景是否达标
