# Purpose
用于每个阶段或每个功能块完成后的交付门禁检查，确保输出结果满足“可运行、可验证、可演示、可截图”的最低交付标准。

## When To Use
适用于：
1. 一个后端模块开发完成后
2. 一个前端页面组完成后
3. 一个移动端流程完成后
4. SQL 和初始化数据完成后
5. 准备进入下一阶段前
6. 准备最终交付前

## Gate Dimensions

### 1. Backend Gate
检查：
- 是否有明确接口
- 是否符合业务规则
- 是否可被前端调用
- 是否说明数据库影响
- 是否有自测方法

### 2. Frontend Gate
检查：
- 页面是否可访问
- 菜单路由是否可达
- 接口绑定是否明确
- 字段是否完整
- 状态和按钮展示是否正确
- 是否适合截图

### 3. Mobile Gate
检查：
- 页面是否适配手机
- 流程是否闭环
- 扫码逻辑是否清晰
- 弱网缓存是否有方案
- 是否适合演示

### 4. SQL Gate
检查：
- 表结构是否完整
- 字段是否与文档一致
- 初始化数据是否足够
- 数据场景是否覆盖预警、缺货、完成态
- 数据是否支撑截图

### 5. Screenshot Gate
检查：
- 登录页
- 首页看板
- 商品管理
- 库存与预警
- 补货建议
- 订单管理
- 波次管理
- 波次详情
- 移动端任务
- 扫码页面
- 完成确认页
- 日志
- 用户管理

## Required Review Output
每次做门禁检查时，输出必须包含：
1. 检查对象
2. 已满足项
3. 未满足项
4. 阻塞项
5. 是否可进入下一阶段
6. 下一步修复建议

## Gate Decision Rules
- 若主链路断裂：不得进入下一阶段
- 若页面不可访问：不得作为可交付页面
- 若 SQL 数据不足以演示：不得进入截图阶段
- 若仅存在非关键样式问题：可带问题进入下一阶段
- 若角色权限不符合要求：视为高优先级问题

## Do Not
- 不要只看代码不看演示效果
- 不要只看页面不看业务链路
- 不要忽略初始化数据质量
- 不要在存在主链路阻塞时给出“可交付”判断
