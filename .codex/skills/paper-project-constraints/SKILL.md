# Purpose
用于约束所有 agent 的行为边界，确保项目始终符合“毕业论文配套实战系统”的定位，不跑偏成企业级复杂系统。

## When To Use
适用于所有开发、规划、审查阶段，默认全程启用。

## Project Positioning
本项目是毕业论文配套演示系统。

项目目标：
- 轻量化
- 业务流程完整
- 界面规范
- 数据可演示
- 便于截图与答辩展示

不是目标的内容：
- 高并发
- 分布式事务
- 微服务拆分
- 复杂机器学习
- 复杂路径规划
- 企业级复杂调度系统

## Mandatory Constraints

### 1. 技术约束
- 后端：Spring Boot + MyBatis-Plus
- 前端：Vue + Element UI / Element Plus
- 移动端：H5
- 数据库：MySQL 8
- 构建工具：Maven
- 定时任务：只能使用 Spring @Scheduled

### 2. 架构约束
- 必须是单体应用
- 前后端分离
- RESTful API
- 不引入微服务架构
- 不引入 MQ、XXL-Job、复杂调度平台

### 3. 业务约束
必须围绕主链路实现：
创建订单 → 生成波次 → 推送拣货 → 扫码确认 → 库存扣减 → 状态更新 → 日志记录

### 4. 算法约束
- 补货建议只使用文档规定公式
- 销量预测只按 7 日移动平均思想处理
- 拣货路径只按库位编号排序
- 不引入复杂算法和模型

### 5. 展示约束
- 页面必须适合论文截图
- 页面字段必须清晰完整
- 风格必须统一
- 数据必须可演示

## Enforcement Rules
在规划、开发、审查时，必须持续检查：
1. 是否引入了不必要的复杂中间件
2. 是否进行了超范围重构
3. 是否擅自改变了业务规则或公式
4. 是否偏离了页面展示与截图需求
5. 是否破坏了“轻量、完整、可演示”的定位

## Required Check Output
如果在某一步使用本 skill 做约束检查，输出至少包括：
1. 当前任务
2. 是否符合论文项目边界
3. 存在的偏离点
4. 修正建议

## Do Not
- 不要允许 agent 自由引入新架构
- 不要允许“为了更先进”而增加复杂度
- 不要允许偏离文档字段和流程
- 不要允许把项目做成不利于演示和答辩的系统
