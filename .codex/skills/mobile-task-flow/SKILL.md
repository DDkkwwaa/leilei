# Purpose

用于规范移动端 H5 作业流程开发，重点保证“能操作、能演示、能跑通、能说明弱网策略”。

## Important Baseline

1. 当前仓库未确认已经存在成熟移动端/H5 工程。
2. 需求文档要求的是目标态移动闭环，不等于仓库现状。
3. 移动端建设应优先兼容现有 `Vue2 + Element UI` 前端工程，走最小侵入式路线。

## When To Use

适用于：

1. 移动端登录页开发
2. 扫码入库页开发
3. 待拣货任务列表开发
4. 拣货详情开发
5. 扫码确认开发
6. 完成拣货开发
7. 弱网暂存与重试逻辑开发

## Flow Requirements

### 1. Login Flow

- 支持作业员登录
- 登录态可持久化
- 失败提示明确

### 2. Picking Task Flow

- 展示任务列表
- 进入任务详情
- 展示库位、商品、待拣数量、已拣状态
- 页面顺序能支撑演示讲解

### 3. Scan Confirm Flow

- 支持摄像头扫码或输入框接收扫码枪数据
- 扫码后做基础校验
- 成功与失败提示清晰
- 记录本次确认结果

### 4. Finish Picking Flow

- 未完成任务不可直接提交完成
- 提交后调用后端接口更新状态
- 成功后返回列表或完成页

### 5. Inbound Flow

- 扫码回显商品信息
- 输入数量
- 提交入库
- 展示清晰结果提示

### 6. Weak-Network Flow

- 支持本地暂存未提交成功记录
- 网络恢复后支持重试
- 能看到同步结果
- 尽量避免重复提交

## Delivery Requirements

每次移动端交付都必须附带：

### 1. Task Goal

本次移动端实现目标。

### 2. Files To Modify

列出本次新增或修改的页面、路由、API、缓存逻辑。

### 3. Flow Covered

本次覆盖了哪一段移动端流程。

### 4. API Binding

依赖哪些后端接口。

### 5. Weak-Network Handling

是否涉及本地缓存、重试、同步状态说明。

### 6. Self Test Path

如何在手机浏览器或模拟器中验证。

### 7. Risks

当前依赖、缺口和风险点。

## Do Not

- 不要把“需求要求 H5”写成“仓库已有 H5”
- 不要把移动端做成复杂 App 方案
- 不要只做静态页面不做流程闭环
- 不要忽略扫码枪输入和弱网说明
