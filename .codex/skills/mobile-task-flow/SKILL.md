# Purpose
用于规范移动端 H5 作业流程开发，重点保证“能操作、能演示、能跑通、能说明弱网策略”。

## When To Use
适用于：
1. 移动端登录页开发
2. 拣货任务列表开发
3. 拣货详情开发
4. 扫码确认开发
5. 完成拣货开发
6. 扫码入库开发
7. 弱网缓存与同步开发

## Flow Requirements

### 1. Login Flow
- 支持作业员登录
- 登录态可持久化
- 失败提示明确

### 2. Picking Task Flow
- 展示任务列表
- 进入任务详情
- 按路径顺序展示商品
- 显示库位、商品、待拣数量、已拣状态

### 3. Scan Confirm Flow
- 支持摄像头扫码或输入框接收扫码枪数据
- 扫码后校验条码
- 成功与失败提示清晰
- 记录本次确认结果

### 4. Finish Picking Flow
- 未完成全部任务不可提交
- 完成提交后调用后端接口
- 成功后更新状态并返回列表或确认页

### 5. Inbound Flow
- 扫码回显商品信息
- 输入数量
- 提交入库
- 结果提示清晰

### 6. Weak-Network Flow
- 本地缓存未成功提交的记录
- 网络恢复后自动同步
- 同步结果可追踪
- 尽量避免重复提交

## Delivery Requirements
每次移动端交付都必须附带：

### 1. Task Goal
本次移动端目标。

### 2. Files To Modify
本次新增或修改的页面、路由、API、缓存逻辑。

### 3. Flow Covered
本次覆盖了哪个移动端流程。

### 4. API Binding
依赖哪些后端接口。

### 5. Weak-Network Handling
是否涉及本地缓存、重试、同步状态处理。

### 6. Self Test Path
如何在手机浏览器或模拟器中验证。

### 7. Risks
当前缺失、依赖和风险点。

## UI Rules
- 适配手机浏览器
- 信息清晰
- 按钮足够大
- 操作路径短
- 页面适合现场演示

## Do Not
- 不要把移动端做成复杂 APP 方案
- 不要忽略扫码枪输入场景
- 不要只做静态页面不做流程闭环
- 不要省略弱网处理说明
